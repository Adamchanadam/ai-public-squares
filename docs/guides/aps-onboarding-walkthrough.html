1.開始之前 — 你會走完點樣嘅一條路

由「兩位用戶各自部機冇接過」開始,行到「第一次跨機交接跑通,雙方可以日常用」為止。總時間約 60–90 分鐘,假設 Drive 已經 mount 好。

Adam / Jay 只係兩個 example 用戶名。呢個 repo 係 通用 APS template,唔綁定特定項目或者特定人。下面教學用「Adam」「Jay」做兩個 narrative protagonist 講故,但實際落地時佢哋係 User A(發起接駁嘅一方)同 User B(對方)— 邊個用都得,你哋自己決定 agent_id、項目代號、工作目錄路徑全部。試演留低嘅 demo workspace 同 demo slug 只係 MVP 驗證嗰陣用嘅 fixture,作為 Bridge Pack 嘅 canonical source,唔係 default value。

五個方格嘅詳細做法,下面段落逐個展開。實線表示要等上一步完成;虛線表示可推遲。

呢份教學寫畀真人睇,口語、narrative、夾實際例。執行步驟嘅精準版本(包括 agent 程式要照住做嘅技術細節)喺 技術版計劃文檔,兩份對讀就最齊。
用語慣例:方括號 placeholder 要 substitute。教學入面所有方括號嘅短語 — 例如 [項目代號]、[題目]、[時間戳]、[工作目錄]、[Jay 部機嘅共享資料夾絕對路徑] — 都係 placeholder,屬 forcing function。複製去寫入規則檔、發 WhatsApp 之前,要逐個換做你哋第 3 節拍板嘅實際值。協定本身通用,教學文字唔列具體 value;具體 value 由你哋自己決定。
Placeholder 樣本 — 形狀同例。每個 placeholder 嘅 substitute value 應該係邊種樣,以下用幾個 hypothetical 例做 reference(下面例都唔係實際 demo 嘅 slug,純粹樣示):
  • [項目代號] — snake_case 短名(英文小寫加底線加數字,長度 30 字內)。例:branding_2026、logo_refresh_q2、internal_pitch、cobeing_website。協定上鎖細楷,Windows NTFS 雖然唔分大細楷,但 Drive 同步去其他系統有機會出 bug,所以 stable 細楷。唔好放中文、空格、特殊符號。
  • [Adam 部機嘅共享資料夾絕對路徑] / [Jay 部機嘅共享資料夾絕對路徑] / [Hub 絕對路徑] — 由磁碟字母開始嘅 absolute path,指向你部機掛載 Hub 嘅實際位置;最後一段必然係 AI_Public_Squares。例:G:\Cloud\AI_Public_Squares、D:\Work\AI_Public_Squares、H:\我的雲端硬碟\Marketing\AI_Public_Squares。雙方各自寫自己部機嗰版,磁碟字母 / 中間段可以唔同。
  • [工作目錄] — 你部機嘅項目工作目錄絕對路徑(裝有 Agent Handoff Kit 嗰個 root,即係你開新對話畀 agent 嗰個 folder)。例:C:\Work\branding、D:\Projects\client_x。
  • [題目] — snake_case 短主題,描述呢個包做緊咩。例:logo_convergence、brand_voice_v2、color_semantics、tagline_review。同樣鎖細楷加底線。
  • [時間戳] — UTC ISO 8601 compact 格式 YYYYMMDDTHHMMSSZ,反映你出包嗰刻嘅 UTC 時間。例:20260520T103000Z(= 2026 年 5 月 20 日 10:30:00 UTC)。Agent 程式自動生成,你唔需要手寫。

  • Drive 桌面版已 sign-in,個共享資料夾右 click 設咗「可離線存取」。
  • Claude Code 裝好,可以喺你嘅工作目錄([工作目錄])開新對話畀 Agent Adam。
  • 該工作目錄入面已裝好 Agent Handoff Kit(AGENTS.md + dev\ 一系列檔案存在)。
  • WhatsApp 可以聯絡到 Jay。
Jay你部機應該已經有
  • Drive 桌面版已 sign-in,見到 Adam share 出嚟嗰個共享資料夾。
  • Claude Code 裝好,可以喺你揀嘅工作目錄開新對話畀 Agent Jay。
  • 你嘅工作目錄入面已裝好 Agent Handoff Kit,doctor 跑得通。
仲未準備好邊樣?呢樣係硬性前置,唔做齊唔好繼續。Drive 未設「可離線存取」,衝突避免嘅 guarantee 都不成立。

3.先傾妥幾件嘢

唔郁任何檔。十五分鐘左右嘅電話或者 WhatsApp 傾妥即可 — 你哋實際要拍板嘅,只有三件;其餘四項都有合理默認,沿用即可。呢一步係用戶之間嘅事,唔涉及兩個 agent 程式。

要拍板嘅三件事

要傾嘅嘢建議備註
項目代號 揀新嘅 snake_case 短名(例:branding_2026、logo_refresh) 揀全新 slug 最乾淨 — 唔需要清任何已有 lane 資料,協定亦唔受影響。若想沿用某個共享資料夾入面已有嘅 slug(例如曾經做過試演或者過往項目用過),要先清舊資料(刪舊 packets、reset outbox)— 多咗一個 step,通常唔抵。樣本嘅完整形狀規則見第 1 節「Placeholder 樣本」。
Jay 部機路徑 Jay 自己揀工作目錄 + 喺自己部機 mount Hub 例:工作目錄 D:\Work\[項目代號]_Jay\(要 kit-init,doctor 跑得通);Hub 磁碟字母可能唔同(例 H:\ 唔係 G:\),呢個合理 — 個小規則檔內各自寫實際路徑,協定本身唔受影響。
大檔試跑要唔要做 建議做,可推遲 第一份真正過 50 MB 嘅資產之前一定要做。如果近期都係文字材料,可以遲啲再補。

預設默認嘅四件事(唔郁就 OK)

項目默認值幾時要改默認
Adam agent 名 自己揀(snake_case,1-30 字內;MVP fixture 用咗 adam 做例) 揀短啲、stable、容易讀就得;唔需要跟 fixture。
Jay agent 名 自己揀(snake_case;MVP fixture 用咗 jay 做例) 同上。
共享資料夾入面已有嘅 lane 資料 留底唔郁(揀新 slug 嘅前提下,已有 lane 自然唔受影響) 只有揀沿用某個已有 slug 嗰陣才有得選清/留;揀新 slug 就唔關事。如果共享資料夾啱啱啟動、未有任何 lane,呢 row N/A。
Adam 部機 Hub 路徑 + Drive 分享權限 Adam 嗰邊部機掛載 Hub 嘅實際絕對路徑([Adam 部機嘅共享資料夾絕對路徑])+ Drive 上層 AI_Public_Squares 資料夾 share 畀 Jay 有寫權 若 Drive sharing 設喺 slug 子資料夾而唔係 AI_Public_Squares root,新 slug 要新加 share。若 Hub 仲未掛載或者 Drive 仲未 share,要先補咗呢兩件嘅機器 setup 先繼續。
雙方傾妥之後

Adam 打開技術版計劃文檔最尾嘅參數表,將拍板嘅嘢同確認嘅默認填齊。呢一刻就係呢一節完成。

4.Adam 接駁自己部機

呢一節 Adam 喺自己部機,叫 Agent Adam 幫手執行;Jay 唔需要喺度。完成之後 Adam 嘅工作目錄已接好個橋接小規則,但仲未交收過任何包。

有 helper script(tools/aps-onboard.ps1)可以一次過落 4 個 file edit(步驟 2-5 嘅機械部分 — Identity 連 4 個 governance row 一齊裝)。詳見技術版計劃嘅 Tooling shortcut 段。步驟 1(read-only baseline check)同步驟 6(SESSION_HANDOFF Durable Anchors,涉判斷)仲係手做。下面手做版本仍然係 authoritative reference,跑 script 之前可以先睇明白每步做緊咩。
Adam同Agent Adam逐步動作

  1. Adam 叫 Agent Adam 做唯讀檢查

    Adam 喺工作目錄開新對話,叫 Agent Adam 做以下事但唔好改檔:列 root 內有冇 AGENTS.md、dev\SESSION_HANDOFF.md、dev\RULE_PACKS.md、dev\PROJECT_INDEX.md、dev\DOC_SYNC_REGISTRY.md;睇 AGENTS.md 頭幾行記低版本;跑 doctor。

    Agent Adam 報告:版本、doctor 通過幾多項、git 狀態。冇任何檔案改動。
    如果版本舊過 0.1.7:Adam 叫 Agent Adam 跑升版指令,doctor 要全部通過先繼續落一步。

  2. 叫 Agent Adam 抄個橋接小規則檔落嚟

    Adam 喺對話入面叫 Agent Adam 將呢個 repo 隨附嘅 demo Bridge Pack(MVP 驗證嗰陣留低,canonical source)抄落工作目錄相同位置 — 路徑類似 [呢個 repo 嘅 sibling]\Demo_Agent_Adam_Public_Squares\dev\rules\aps-bridge.md(demo Adam fixture — procedural 部分同 demo Jay pack 一樣,只 Identity 唔同)。除咗最上面個 Identity 段之外,內容唔郁。

    Identity 段改成 Adam 部機嘅實際值:

    - agent_id: adam
- project_slug: [項目代號]
- hub_root: [Adam 部機嘅共享資料夾絕對路徑]
- other_agent_id: jay

    複製出嚟之後,將 [項目代號] 換做第 3 節拍板嗰個值(例 branding_2026);[Adam 部機嘅共享資料夾絕對路徑] 換做你部機掛載 Hub 嘅實際 absolute path(例 G:\Cloud\AI_Public_Squares),先寫入規則檔。完整形狀規則見第 1 節「Placeholder 樣本」。

    將工作目錄嘅小規則檔同試演嗰份比較,差異只係 Identity 段。

  3. 叫 Agent Adam 喺路由表入面註冊呢個小規則

    喺 dev\RULE_PACKS.md 嘅路由表加多一行(## Routing Rule 之前),叫 Agent Adam 抄試演嗰行原文。

  4. 叫 Agent Adam 將共享資料夾登記做 External Source

    喺 dev\PROJECT_INDEX.md 嘅 External Sources 表加一行,將共享資料夾路徑同寫權範圍寫低。

  5. 叫 Agent Adam 喺同步登記表加兩行

    喺 dev\DOC_SYNC_REGISTRY.md 加兩行(「出包」一行 + 「消化登記」一行),抄試演原文即可。

  6. 叫 Agent Adam 更新交接檔嘅 Durable Anchors

    喺 dev\SESSION_HANDOFF.md 嘅 Durable Anchors 同 Active Objective 各補一句,等將來開新對話嘅 Agent Adam 一讀就知工作目錄已接通共享資料夾,啟動同收尾要跑嗰兩步。

    dev\SESSION_HANDOFF.md 入面搜 "APS" 有至少兩個位提到。

所有改動以小 commit 形式記低,標題用 feat(aps):(新檔)或 docs(aps):(governance 改動)前綴。Agent Adam 寫 commit message,Adam 確認後 commit。

呢一節完之後:Adam 工作目錄已接好,但因為 Jay 嗰邊未接通,任何試出包都未有 receiver。先等 Jay 完成第 5 節,然後第 6 節雙方一齊跑。

5.Jay 接駁自己部機

Jay 嗰邊都可以用同一個 helper script(tools/aps-onboard.ps1)(Adam 整完之後用 WhatsApp / Drive 連同 demo Jay pack 一齊傳過嚟畀 Jay 跑)。詳見技術版計劃嘅 Tooling shortcut 段;Jay-side invocation 用 -AgentId "<your_agent_id>" -OtherAgentId "<counterpart_agent_id>"(以本教學嘅 narrative 為例,分別係 jay 同 adam),demo Jay pack 做 -DemoPackPath。下面手做版本仍然係 authoritative。

呢一節 Jay 喺自己部機,叫 Agent Jay 幫手執行。Adam 嘅角色係預先 WhatsApp 將試演 Jay 嗰份小規則檔同三行參數 send 畀 Jay,然後隨時答問題。Adam 同 Agent Adam 都唔可以代 Jay 跑命令。

Adam預先準備

將兩件嘢 send 畀 Jay:

  • 呢個 repo 隨附嘅 demo Bridge Pack(Jay-side fixture),路徑類似 [呢個 repo 嘅 sibling]\Demo_Agent_Jay_Public_Squares\dev\rules\aps-bridge.md — 同 Adam-side 嗰份 procedural 內容一樣,只係 Identity 唔同。
  • 一張短紙條,記第 3 節拍板嘅七行參數結果。
Jay同Agent Jay逐步動作

  1. Jay 自己跑部機前置 checklist

    逐項打剔:Drive 桌面版 sign-in 咗、共享資料夾見到、共享資料夾已設「可離線存取」、工作目錄揀好兼 kit-init、doctor 跑得通。

    做完一句 WhatsApp 答 Adam,例:checklist OK · 工作目錄: D:\Work\[項目代號]_Jay(將 placeholder 換做你部機嘅實際路徑)。

  2. Jay 叫 Agent Jay 接駁小規則檔

    將 Adam send 嚟嗰份 save 入 [工作目錄]\dev\rules\aps-bridge.md,叫 Agent Jay 改 Identity 段:

    - agent_id: jay
- project_slug: [項目代號]
- hub_root: [Jay 部機嘅共享資料夾絕對路徑]
- other_agent_id: adam

    複製出嚟之後,將 [項目代號] 換做第 3 節拍板嗰個值(例 branding_2026);[Jay 部機嘅共享資料夾絕對路徑] 換做你部機實際嘅路徑(例 D:\Work\AI_Public_Squares;磁碟字母可能係 H:\、D:\ 或其他)。完整形狀規則見第 1 節「Placeholder 樣本」。

    跟住叫 Agent Jay 做同第 4 節第 3–6 步對等嘅動作:路由表加行、External Source 加行(寫權範圍寫「永不寫 from_adam 同 _ack\adam.ack.json」)、同步登記表加兩行、交接檔更新 Durable Anchors。

呢一節完之後:雙方工作目錄都接好,但共享資料夾上仲未有真嘅包過。第 6 節就係跑首次跨機交接。

6.首次跨機交接

由 Adam 出第一個真包開始,經 Agent Adam 寫入 Hub,Adam 一句 WhatsApp 通知 Jay,Jay 叫 Agent Jay 撿包,Jay 做完叫 Agent Jay 出回包,Jay 一句 WhatsApp 通知 Adam,Adam 叫 Agent Adam 撿到回包收結。揀一個細嘅內容(一段品牌語氣 note、一個 logo spec 段落)做第一次,唔好試大檔(嗰個係第 7 節)。

Adam 用戶 Agent Adam 程式 Agent Jay 程式 Jay 用戶
敏感資料禁區。Packet 入面唔放 credentials、API key、未公開財務、未授權人名。共享資料夾屬中立區,唔適合留 secret。要交付敏感資料,行另一條 secure channel(WhatsApp 私訊、1Password share 等),packet body 入面用抽象 reference 指過去,唔貼實際值。
用戶決定,Agent 執行 — 之間有一格。Agent Adam 同 Agent Jay 撿到包之後,係 propose 待辦畀對應嘅用戶睇,唔自動修改本地 SSOT 嘅檔。用戶揀做唔做、揀幾時做、揀做幾多,先叫個 agent 動手執行。Packet body 講「請更新 X」唔等於對方 agent 自動更新 X — 「propose → 用戶拍板 → execute」係故意保留嘅人手 step,等用戶控住每次工作流。
首次跨機驗證 vs 日常運作。下面嘅完整十步係跨機接駁計劃第一次跑做嘅動作 — 包括 UTC 時間記錄(量度 Drive 同步延遲)同 WhatsApp 完整模板。呢啲屬一次性驗收,跑通之後就有 baseline。日常運作精簡好多 — WhatsApp 短訊一句、receiver 對住現有對話講 check Hub 就得,詳見 §8。

四條 lane,實邊框係人,虛邊框係程式。時間由上至下,十步走完一輪。藍色屬 Adam 域,橙色屬 Jay 域。

Adam同Agent Adam· 出第一個真包(步驟 ①–③)
  1. Adam 喺工作目錄開新對話,寫好想交畀 Jay 嘅內容。
  2. Adam 對話收尾,叫 Agent Adam 跟收尾流程出包:Agent Adam 喺共享資料夾下 [項目代號]\from_adam\packets\[時間戳]__[題目]__v1\ 寫 packet.md;append 一行入 from_adam\outbox.log.md;本地 SESSION_LOG.md 加同步行。
  3. (首跑驗收一次性) Adam 自己記低出包嗰刻嘅 UTC 時間,落後計 Drive 同步延遲做接駁計劃嘅驗收。日常 round-trip 唔需要做。
  4. Adam 自己發 WhatsApp 畀 Jay,貼下面呢句([項目代號] 換做實際值)。WhatsApp 唔由 Agent Adam 發 — 係 Adam 親手發:
APS Hub has new traffic. Open a new session and paste this to your agent:
"Read <hub_root>/[項目代號], follow APS PROTOCOL, process unconsumed items in from_adam."

複製出嚟之後,將 [項目代號] 換做第 3 節拍板嘅實際值(例 branding_2026),先發出去畀 Jay。<hub_root> 唔需要換 — 個字 literal 留低,Jay 嘅 Agent Jay 會用自己部機嘅實際 hub_root 解讀。

Jay同Agent Jay· 撿包、做、出回包(步驟 ④–⑧)
  1. Jay 收到 Adam WhatsApp,等 Drive 同步少少時間(通常秒到一分鐘,慢嘅話五分鐘以內都正常)。
  2. Jay 喺工作目錄開新對話,將 Adam 嗰句模板貼落畀 Agent Jay。
  3. Agent Jay 跟啟動流程:讀 from_adam\outbox.log.md + 自己 _ack\jay.ack.json,差集計算 → 列出一個待辦項:Adam 嗰個包,連埋包入面寫嘅內容範圍同要做嘅項目。
  4. 留意:Jay 冇喺呢個新對話解釋過任何上下文,Agent Jay 自己讀晒。
  5. Jay 做工作。完成之後叫 Agent Jay 跟收尾流程出回包:寫 from_jay\packets\[時間戳]__[題目]_reply__v1\packet.md;append from_jay\outbox.log.md;update _ack\jay.ack.json(加一行記低消化咗 Adam 嗰個包)。

APS Hub has new traffic. Open a new session and paste this to your agent:
"Read <hub_root>/[項目代號], follow APS PROTOCOL, process unconsumed items in from_jay."

同上 — [項目代號] 換做第 3 節拍板嘅實際值(例 branding_2026);<hub_root> literal 留低。

detection 順利、Drive 延遲少過五分鐘、共享資料夾冇衝突檔。Jay 嘅 SESSION_LOG.md 加返同步行。
Adam同Agent Adam· 收結整個交接(步驟 ⑨–⑩)
  1. Adam 等 Drive 同步,喺工作目錄開新對話,將 Jay 嗰句模板貼落畀 Agent Adam。
  2. Agent Adam 跟啟動流程撿到 Jay 嗰份回包,讀埋內容範圍同回包內容,呈現畀 Adam。
  3. Adam 叫 Agent Adam 做兩件事:(a) update _ack\adam.ack.json 加一行記低消化咗 Jay 嘅回包;(b) append 一行 close 事件入 from_adam\outbox.log.md,收結 Adam 當初出嗰個包:
[時間] | close | [時間戳]__[題目] v1 | reason:jay ack'd and replied
  1. Adam 再開一次新對話試。Agent Adam 啟動流程應該報「冇待辦」—— 因為個 close 已經將整個 thread 由計算除咗。Agent Jay 嗰邊同樣係冇待辦狀態。
雙方消化記錄表各有一行;雙方事件記錄都有對應出包行(Adam 嗰邊仲多一行 close);共享資料夾無衝突檔;之後再開新對話兩個 agent 都係「冇待辦」。整個交接完成。
頭嘅總時間:Adam 由出包嗰刻到收結嗰刻嘅 wall-clock,試演同機係即時嘅,真實跨機會包括兩次 Drive 同步 + Jay 嘅工作時間,可能幾個鐘到幾日唔等。Jay 幾時開新對話畀 Agent Jay,係由 Jay 用戶決定,唔係 Agent Jay 主動 poll。

7.大檔過嘅試跑(可選)

驗證「過 50 MB 嘅檔唔擺包入面,改放外面再喺包入面講邊度搵」呢條規矩。試演冇試過,第一個真大檔之前一定要試一次。

Adam同Agent Adam動作
  1. Adam 自己準備一個過 50 MB 嘅 dummy 檔。可以係 padded zip、真大 PSD,或者 PowerShell 跑 fsutil file createnew test.bin 62914560 整一個 60 MB 嘅空檔。
  2. Adam 自己將個檔放入共享資料夾嘅 sibling 位:[共享資料夾]\_external\[題目]\[檔名](_external\ 唔存在嘅話就建立)。
  3. Adam 叫 Agent Adam 出個包,包入面 YAML 嘅 ssot_refs: 寫返條絕對路徑:
ssot_refs:
  - "[Hub 絕對路徑]\\_external\\[題目]\\big_asset.psd"

複製出嚟之後,將 [Hub 絕對路徑] 換做你部機掛載 Hub 嘅實際 absolute path(例 G:\Cloud\AI_Public_Squares),[題目] 換做呢個包嘅 snake_case 短主題(例 logo_convergence);\\ 雙反斜線係 YAML escape,唔好改成 single backslash。

  1. 包入面正文寫清楚:檔係咩、點解放外面、Jay 應該點開。Adam 寫,Agent Adam 入檔。
  2. Agent Adam 收尾報告標明:「Externalized: big_asset.psd (XX MB) at [Hub 絕對路徑]\_external\…」。
  3. Adam 自己 WhatsApp Jay 同正常包一樣。
Jay同Agent Jay收方
  1. Jay 正常開新對話、貼模板。Agent Jay 啟動流程列出個包,同時要報埋外置檔嘅路徑畀 Jay 睇。
  2. Jay 親手喺自己部機 Drive 用 Explorer / 設計工具開個外置檔,確認檔可讀。Agent Jay 唔負責呢一步,因為佢只讀寫 Hub 入面嘅檔。
包入面 attachments\ 冇任何檔過 50 MB;外置檔嘅通知出現喺 Agent Adam 收尾報告同 Agent Jay 啟動報告;Jay 可以用 Drive mount 開到個外置檔。

8.日常點用

第 4 至 6 節跑通,APS 已上線。日常用法就係一個重複拍子,每次來回都係呢個十步。每個 pill 標明係用戶動作定 agent 動作。

十步打完即係一條交接收結。實邊框係人,虛邊框係程式。一日唔一定行齊十步,可以中間 pause(例:Jay 慢慢做)。

多條交接同時跑都係兩條呢個十步並行,共享資料夾入面嘅事件記錄同消化記錄表會記晒。

日常 WhatsApp 短訊版。日常運作,WhatsApp 嗰句唔需要再貼完整模板畀對方再 paste 落 agent。一句「Hub 有新嘢」夠數通知對方;對方喺自己工作目錄,對住現有對話講一句「check Hub」即可,Agent 即刻撿包(呢個係默認嘅日常 primary pattern)。開新對話 + paste 整段模板係 fallback,只限第一次跨機(§6)或者你想 fresh start。完整英文模板(§6 嘅 APS Hub has new traffic...)保留畀第一次跨機交接做啟蒙教學;日常跑通之後一句「check Hub」就夠。
頻率估計:Adam → Jay 方向通常一日一兩次。Jay 回包可能即日內,可能等明朝。一條交接嘅 wall-clock 大約 12–48 小時。緊急嘅交接,WhatsApp 加一句「即刻」就得,協定本身唔阻你即時催。

9.中間想交換,唔等對話收結

第 6 同 8 節入面講「對話收尾叫 agent 出包」,呢個係順手做嘅時機,唔係硬性。Adam 或者 Jay 嘅對話中間任何一刻,只要內容 ready,都可以即時交換。

Adam同Agent Adam· 想中途即時出包

Adam 喺對話中間講「我想而家 send 一份畀 Jay」,叫 Agent Adam 依出包流程做。Agent Adam 即刻寫包入共享資料夾 + 加事件記錄。包係不可變嘅 snapshot,Adam 之後喺對話入面做嘅工作,唔會倒灌入已出嘅包。

Adam 自己即刻發 WhatsApp 模板畀 Jay。然後繼續做下面嘅工作。

Jay同Agent Jay· 想中途撿啱啱到嘅包

Jay 喺對話中間收到 Adam WhatsApp,唔需要 close 個對話開新嘅。Jay 對 Agent Jay 講一句官方觸發詞:

  • check Hub — 官方觸發詞,最短最直接。Agent 認 case-insensitive(check hub / Check Hub 都得)。

另外兩句同義講法 — 睇下 Hub 有冇新嘢、未消化 — Bridge Pack 都認,做 fuzzy 兜底(句入面含 Hub 連同 新 / 未消化 / check 任一個字眼,即觸發同一個 handler)。用慣邊句都得,但官方一句係 check Hub。

觸發之後 Agent Jay 即刻去共享資料夾撿,計算差集,將新待辦呈現畀 Jay。Jay 揀做唔做、幾時做。

呢個係手動觸發,唔係自動推送。Agent Jay 唔會主動去 poll 共享資料夾 — 佢只係喺 Jay 開新對話啟動嗰陣、或者 Jay 中間叫一句嗰陣先去 check。換言之,Jay 唔開對話 / 唔叫,Agent Jay 永遠唔知有新嘢。
真係想自動推送(部機自動彈通知)?呢樣要本地 daemon 監察共享資料夾入面 outbox 嘅修改時間,出咗純檔案範圍嘅設計,加緊耦合 dependency。設計取捨上,WhatsApp 嘅人手叫一聲係故意保留,等收嘅人控制咩時候被打斷;自動推送嘅 cost 係噪音同中斷工作流。值唔值要視乎你哋實際用嘅頻率同緊急程度,Phase 4 落地之後睇返再決定。

10.出咗錯點救

10.1 出咗 Drive 衝突副本(file (1).md 之類)

理論上唔應該發生,因為單寫道規矩。Bridge Pack 喺每次啟動同 check Hub 觸發嗰陣會自動掃共享資料夾,掃到任何含 conflict 字眼嘅檔即時報畀你睇 — 用戶唔需要主動跑命令查。如果 Agent 報咗,或者你自己肉眼見到:

  • Adam 或 Jay 唔好即時手動刪。
  • 停手,寫低嗰個衝突檔嘅完整路徑同名。
  • 對返單寫道規矩,搵出邊個 agent 程式寫咗對方通道嘅檔(通常嗰個就係根源 — Agent Adam 寫咗 from_jay,或者 Agent Jay 寫咗 from_adam)。
  • 修返根源之後,先決定保留邊個版本(通常係原寫者嗰份),衝突副本移走或者更名做 .investigated。

10.2 出咗包但內容錯 / 想撤回

用戶發現錯,用戶 + agent 一齊 check 對方有冇登記,然後用戶決定,agent 執行。

撤回嘅一行寫法(由 agent 寫):

[時間] | withdraw | [時間戳]__[題目] v1 | reason:作者要求撤回原版

10.3 個 agent 撈錯通道(寫咗對方通道、寫咗對方消化記錄表)

邊界硬規矩應該攔到。如果真係漏咗:

  • 用戶即刻停手,唔好繼續叫 agent 出包。
  • 用戶檢查嗰個誤寫究竟係 placeholder(可清)定真內容(由用戶決定點處理)。
  • 用戶決定是否手動清除誤寫;事件 + 原因記入 SESSION_LOG.md(由 agent 寫),等下次開新對話可以追溯。

10.4 忘記收結,thread 永遠掛緊待辦

冇大件事。下次任何一邊開新對話,個 agent 啟動流程列佢做待辦,用戶叫 agent append 一行 close 事件就 OK。close 完之後再開新對話就係冇待辦狀態。

10.5 Drive 同步延遲過五分鐘

幾乎一定唔係協定問題。Adam 或 Jay 用戶順序檢查:

  1. Drive 桌面版係咪 running、係咪 sign-in 緊?
  2. 個共享資料夾係咪設咗「可離線存取」?如果只係「串流」可能會延遲。
  3. 網絡頻寬足夠?大檔嘅話可能要再等。
  4. 對方部機係咪 online?Drive 唔會主動 push,要 receiver 嗰邊 Drive 客戶端 sync 先見到。
