由你部機未接過 APS 嗰一刻起,行到你同 Jay 第一次跨機交接完成。寫畀真人讀。你同 Jay 兩個角度,各自跟住做就得。

之後會逐步加

每一份對應一個落地之後嘅實際情景。會跟住項目進度慢慢補。

  • 接落嚟嘅日常用法 — 第一次跨機接通之後,每日點開新對話、Jay 收到通知後做乜、出包同收包嘅重複動作。
  • 大檔點過 — 設計圖、影片過大,唔好擺入交接包入面。要另外擺一個位,再喺包入面講邊度搵。呢份講具體做法。
  • 協定升版嘅做法 — 雙方覺得規矩要改、要加欄、要動流程,點先有秩序咁過版本,留底等將來追溯。
  • 多一個人入嚟協作 — 三個或以上嘅人同時做一個項目,點開新通道,點計算多方嘅待辦。
  • 換另一個 AI 程式 — 由 Claude Code 轉去其他同類程式之後,邊啲位要重新驗,會唔會出現唔同行為。
  • 出咗重複檔點救 — 萬一真嘅出咗 Drive 衝突副本,點順序處理,點搵原因,點先唔好越救越亂。
  • 用 Notion 做進度看板 — 將活動同消化記錄單向同步去 Notion,等人手一眼睇晒邊單對邊單。

呢個教學中心會跟住每一階段落實,慢慢補新教學。
