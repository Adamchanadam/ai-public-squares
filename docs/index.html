AI Public Squares

兩個人,各自在自己的電腦上,以 AI 共同製作一份品牌指引、撰寫一份報告、設計一個項目 — 但檔案來去往返、上下文重複交代?

這個工具是什麼

兩個人在兩部電腦上 — 可能是朋友、同事,或是隔著一個太平洋的協作夥伴 — 以 AI 共同完成一份品牌指引、撰寫一份報告、設計一個項目。最常遇到的問題:

📦
「最新版的檔案在哪裡?」
每次更新都要透過 WhatsApp 通知一次,Google Drive 一邊更新一邊覆蓋。
🤖
「我這部 AI 記不起我們之前的討論」
對方開啟新 AI 對話時,須重新交代上下文,費時費力。
⚠️
Google Drive 出現衝突副本
雙方同時儲存,Google Drive 產生「(1).md」或「(conflict).md」,無法判斷哪一份是最新版。
🔁
每次往返均須重做
對方回覆之後又要重複一遍:搬運檔案、解釋背景、再次經歷整個流程。

本工具的設計:Google Drive 上一個共享資料夾,加上一套寫入規矩 = 雙方電腦的 AI 自動知道對方的進度;新內容自動呈現給使用者;衝突由結構性避免。沒有伺服器,沒有額外的雲端帳號,沒有自動推送通知(刻意如此設計),純粹依靠 Google Drive 上的檔案結構。

安裝前置事項

本工具有兩項先決條件,須先完成。

第一項 — 安裝 Agent Handoff Kit(治理基礎)

本 repo 與 Agent Handoff Kit 互相配合。Agent Handoff Kit 提供基本的交接、紀錄、健康檢查機制;APS 在此基礎上加入跨機協作的功能。

未先安裝 Agent Handoff Kit,APS 的檔案結構將無法獨立運作。

前往 https://github.com/Adamchanadam/agent-handoff-kit,依其指引安裝。一條命令即可;命令列印寫入計劃後,檢查路徑無誤,再輸入 yes 確認寫入:

$npx --yes @adamchanadam/agent-handoff-kit@latest init

第二項 — 基本前提

  • 電腦需安裝 Google Drive 桌面版(前往 https://www.google.com/drive/download/ 下載),登入之後將共享資料夾設為「離線存取」
  • 電腦需安裝 Claude Code(前往 https://claude.com/code 下載)或 Codex
  • 與你的協作夥伴之間能透過 WhatsApp 通訊

以上任何一項尚未完成?各個官方網頁均有逐步指引。先安裝完成,再回來執行下一步。

目前可以怎樣試

npm latest 0.2.4 pre-release 的主路徑是先安裝套件,再用互動式 aps init 檢查與建立初始 Hub skeleton、本地設定與短命令日用流程。0.2.4 保留 revise、withdraw 與 doctor。正式跨機使用前,每個真實項目仍要各自驗證 Google Drive 同步。

$npm install --save-dev @adamchanadam/aps
$npx aps init
  1. npx aps init 會逐步問你 Hub 路徑、項目代號、雙方 agent id 與角色,並解釋每個值的用途;Hub 路徑指你電腦上 Google Drive 同步出來的 AI_Public_Squares 資料夾完整路徑。你可按 Enter 接受建議值。
  2. 工具會列出寫入計劃,包括技能安裝位置、Hub 項目資料夾與本地 .aps/config.json 路徑。
  3. 確認無誤後輸入 yes 才真正寫入。之後日常命令可以縮短為 npx aps inbox、npx aps doctor、npx aps publish --topic ... --body ...。
值你要填甚麼例子
hub-root Google Drive 桌面版同步到本機的共享資料夾實際路徑。請在檔案總管打開該資料夾,複製地址列路徑;不要輸入 G:\...\AI_Public_Squares 或任何含 ... 的省略寫法。 G:\我的雲端硬碟\AI_Public_Squares
project 這次協作項目的短代號。只用英文小寫、數字、底線;不要用中文或空格。第一次測試可用新的測試名。 aps_uat
agent-id 你這邊 AI 的名稱。發起方通常用自己的短名。 adam
other-agent-id 對方那邊 AI 的名稱。對方設置時會和你的 agent-id 對調。 jay

若你只是單機試跑,可先用一個本機測試資料夾做 hub-root;若你要和協作夥伴真正跨機試用,必須使用雙方都可同步到的 Google Drive 共享資料夾。

進階用法:若由 AI 或腳本代為執行,可使用非互動命令 npx aps init --hub-root ... --project ... --agent-id ... --other-agent-id ... --role A|B。方括號與 ... 不可照抄;CLI 會在執行前攔截這類假路徑。

設置完成後,工具會把 APS 技能安裝到 Claude Code / Codex,建立 Hub skeleton、Bridge Pack 與 starter pack。自然語言日常操作仍在硬化中;底層 CLI 已可支援以下方向:

  • 「我有嘢俾對方」 → 工具自動將內容打包,生成一句 WhatsApp 通知句供你發送
  • 「對方嗰邊有冇新嘢?」 → 工具自動從共享資料夾擷取對方的檔案,列出待辦
  • 「Google Drive 同步唔到」 → 工具偵測問題並提出修復方法
最終目標是全程無需先閱讀任何技術文檔。目前仍處於前期測試階段,公開頁先呈現 CLI 優先試行路徑與手動備用參考。
目前狀態:npm latest 0.2.4 pre-release 提供 bridge-pack、互動式 aps init 技能安裝、初始 Hub skeleton、Bridge Pack、starter pack、本地設定與短命令日用流程。CLI 已有最小 publish / inbox / consume / close 指令,並新增 revise / withdraw / doctor。本機互動式設定回歸已通過,並沿用一次維護者真實 Google Drive 跨機往返驗證作為協定證據。但仍未等同自然語言日常操作或補救流程已產品化。請勿把本頁當成已可生產使用的安裝說明。

本工具的構想 — 5 個核心設計

想理解背後如何運作 — 無需先閱讀即可使用,但對技術背景的讀者具參考價值。

1
Google Drive 上一個雙方都見到的共享資料夾
雙方 AI 程式都視之為本地資料夾直接讀寫,完全無需 API 或雲端介接機制。Google Drive 預設的同步機制已經足夠。
2
各自寫各自的子資料夾,不寫對方的
User A 的 AI 僅寫 User A 那邊,User B 的 AI 僅寫 User B 那邊。Google Drive 同步衝突由結構性避免,不靠人手紀律。
3
一旦發出的檔案就鎖定,需修改則出新版本
每個交接包寫入 Google Drive 之後即不再修改;要更新 = 出新版本並標明取代哪一版。雙方永遠引用清晰的版本,不會誤引到對方未定稿的中間狀態。
4
協作夥伴開啟新對話時,AI 自動擷取你的新內容
User B 開啟新 Claude Code 對話時,其 AI 啟動讀序中已內嵌「前往 Google Drive 查看 User A 是否留下新檔案」,自動列出待辦 — User B 無需解釋上下文。
5
日常 WhatsApp 短訊 + 一句觸發句
User A 發出內容之後,WhatsApp 一句「Hub 有新嘢」;User B 對著現有 AI 對話講「check Hub」,AI 立即擷取。無需貼任何模板,沒有自動推送通知(刻意如此 — 由你決定何時被打斷,而非由 AI 主動發送提示)。

本項目如何驗收

本項目以「全面檢」守住三件事:公開承諾一致性、發佈前可信度、協定實際運行正確性。意思是:人類入口寫了什麼,CLI 與橋接規則包便要支撐到什麼;尚未完成者必須清楚標示為前期測試或目標體驗。

  • 公開承諾一致性:本頁、README、教學中心、完整落地教學、CLI help 與 skill spec 不可互相矛盾。
  • 發佈前可信度:README、GitHub Pages、npm package、橋接規則包與教學頁在外發前須能讓外部讀者按當前狀態理解與操作。
  • 協定實際運行正確性:Hub、單寫通道、交接包、收件記錄、收結 / 撤回與啟動時待處理計算須有實際操作記錄;未能驗證者須清楚標為受阻。

全面檢亦須走通四條主要使用流程:零認知讀者入口流程、手動設置流程、日常協作流程、出錯補救流程。具體規則見 分層 QC;AI / 維護者層真源為 dev\qc\triggers.md。
