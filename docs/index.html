AI Public Squares

兩個人各自喺自己部機、用 AI 工具合作整一個項目 — 但 file 黎黎去去成日撞、context 講完又要再講?

v0.1.0 placeholder 完整 init 後台起緊 建立 2026-05-20 最新更新 2026-05-22

呢個工具係咩

兩個人喺兩部機 — 可能係朋友、同事,或者隔住一個太平洋嘅 partner — 用 AI 工具一齊整一份品牌指引、寫一份報告、做一個 design。最常撞嘅問題:

📦
「最新版啲檔喺邊?」
每次 update 都要 WhatsApp send 一次,Drive 一邊整一邊覆。
🤖
「我嗰部 AI 唔知我哋之前傾過咩」
對方開新 AI session 要從頭講上下文,費時費力。
⚠️
Drive 整咗 conflict file
兩邊同時 save,Drive 出個「(1).md」 /「(conflict).md」,唔知 keep 邊個。
🔁
每次來回都重做一次
Partner 回包又要倒返,搬檔、解釋、再講一次。

APS 嘅設計:Drive 上一個 shared folder + 一套寫入規矩 = 兩部機嘅 AI 自動知道對方做緊咩、新嘢自動 surface、衝突結構性避免。冇 server、冇外加帳號、冇 push notification(故意),純粹 Drive 上嘅 file。

點開始

目標體驗(完整可用 timeline 見「而家狀態」):

$npx @adamchanadam/aps init
  1. 喺你工作目錄(任何 folder)跑上面嗰條命令。
  2. 工具 ask 你幾條 plain 廣東話 / 英文問題:用咩 Drive folder、項目叫咩名、partner 嘅名等。
  3. 跑完之後,喺 Claude Code 對話入面直接講 plain words:
    • 「我有嘢俾 Jay」 → 工具自動出包 + 生成 partner WhatsApp 通知句
    • 「Jay 嗰邊有冇新嘢?」 → 工具撿 partner 嘅檔 + 列待辦
    • 「Drive 同步唔到」 → 工具偵測 + 提出 recovery 方案
全程冇需要 APS protocol knowledge。工具自己處理 Drive 入面嘅檔案、版本、待辦、衝突避免。

工具背後嘅 5 個 design 招數

想理解背後點 work — 唔需要先讀就用得到,但對技術背景嘅讀者有 reference value。

1
Drive 上一個雙方都見到嘅 shared folder
兩邊 AI 程式都當佢係本地 folder 直接讀寫,完全唔需要 API / 雲端 connector。Drive 預設嘅 sync 已經足夠。
2
每邊各自寫各自嘅 sub-folder,唔寫對方嘅
User A 嘅 AI 淨係寫 User A 嗰邊,User B 嘅 AI 淨係寫 User B 嗰邊。Drive 同步衝突由結構性避免,唔靠人手紀律。
3
一發出嘅檔就鎖,要改 = 出新版本
每個交接包一寫入 Drive 就唔再改;要 update = 出 v2 + 註明取代邊一版。雙方永遠引到清晰版本,唔會誤引到對方未定稿嘅中間狀態。
4
Partner 開新對話嗰陣,個 AI 自動撿你嘅新嘢
User B 開新 Claude Code session 嗰陣,個 AI 啟動讀序入面已經 baked 入「去 Drive 睇下有冇 User A 留低嘅新檔」,自動列待辦 — User B 唔需要解釋上下文。
5
日常 WhatsApp 短訊 + 一句 trigger
User A 出完嘢,WhatsApp 一句「Hub 有新嘢」;User B 對住現有 AI 對話講「check Hub」,個 AI 即刻撿。冇需要 paste 任何模板,冇 push notification(故意 — 你決定幾時被打斷,唔係 AI 主動 ping)。
