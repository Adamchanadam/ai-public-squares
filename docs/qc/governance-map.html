分層 QC — 觸發詞彙

三 tier vocabulary 將既有 6 層 QC mechanism 收口成 3 個 trigger entry point。

本卡片：reference SSOT：dev/qc/triggers.md 建立：2026-05-21 S6

設計意圖

過往「全套 QC / 完整 QC / 跑驗收 / 整體 audit」此類含混詞令人類與 AI 對驗收 scope 之理解漂移。三 tier vocabulary 鎖定三件事:每個 tier 之覆蓋與時間 explicit,outer 自動含 inner,含混詞觸發停手反問。

本卡片只屬日常閱讀與操作參考;trigger keyword、tier 定義、覆蓋範圍之 canonical 真源於 dev/qc/triggers.md。如本 HTML 與 SSOT 有 drift,SSOT 文件 wins。

三 tier 速覽

Tier觸發 keyword時機時間預算
🟢 快檢 跑快檢 / quick-check / qa-gate session 收工前 / commit 之前 < 2 分鐘
🟡 外發前檢 跑外發前檢 / release-check / handover-verify push remote / 分享 plan / 跨 workspace 傳檔之前 5–10 分鐘
🔴 全面檢 跑全面檢 / full-check / full-audit 跨機 round-trip / PROTOCOL.md 改版 / kit major upgrade 之前 30+ 分鐘

三 tier 詳述

🟢 快檢

別名：跑快檢 / quick-check / qa-gate

時機：session 收工前；五區段執行完之後；commit / amend 之前。

覆蓋（4 項）：

  • kit doctor 報 status: passed
  • git status 同 handoff Workspace Identity 一致
  • 當次 session 所有 grep acceptance 全部命中
  • 有 script 改動：parse-check + DryRun against demo

驗收：四項全過。
失敗：修源重跑。

🟡 外發前檢

別名：跑外發前檢 / release-check / handover-verify

時機:物件離開本 workspace 之前(push remote / 分享 plan / 傳 Bridge Pack 予 Jay)。

覆蓋：快檢 4 項 + 5 項：

  • 跨工作目錄 placeholder audit
  • plan / 教學頁 / Bridge Pack 三處 cross-read
  • HTML preview 視覺確認
  • PII / secrets sweep
  • Voice / terminology discipline 掃描

驗收：9 項全過。
失敗：handover stop。

🔴 全面檢

別名：跑全面檢 / full-check / full-audit

時機：跨機 round-trip / 協定升版 / 引入新 agent / kit major upgrade 之前。

覆蓋：外發前檢 + 5 項人工 / 跨 workspace：

  • Class-C 跨 workspace audit
  • MVP-style round-trip regression
  • Bridge Pack startup 行為 trace
  • 五區段 / 自審 retrospective
  • Audit report 輸出

驗收：14 項 + report committed。
失敗：逐條列阻擋項。

嵌套規矩

三 tier 嚴格嵌套：

  • 跑外發前檢 必自動跑 快檢 4 項先
  • 跑全面檢 必自動跑 外發前檢(即連 inner 快檢都跑埋)

AI 不可於被 outer trigger 觸發後省略 inner layer。

反問規矩

當用戶用以下含混詞，AI 必須停手反問「你指快檢 / 外發前檢 / 全面檢?」：

含混詞對應動作
全套 QC反問
完整 QC反問
做 QC反問
跑驗收反問
Audit / 整體 audit反問
QC(單字)反問

未列於以上 list 之含混表達,按字面 + context 解;如仍有 ambiguity 寧反問。

典型用法

  • 🟢 快檢:commit 前 spot-check;五區段執行完之後之 baseline 驗證;改完 doc 試 anchor 不斷
  • 🟡 外發前檢:push remote 之前;Adam 傳 Bridge Pack 予 Jay 之前;新 Phase plan 分享予外人之前
  • 🔴 全面檢：跨機 round-trip 落地之前；協定升版之前；引入新 agent 之前；連跑 N 個 session 後重置 baseline

點呼叫

對住 AI 講一句 trigger keyword 即可：

  • 「跑快檢」— AI 跑 doctor + git status + grep set
  • 「跑外發前檢」— AI 跑快檢 + 五項外發 audit
  • 「跑全面檢」— AI 跑外發前檢 + 五項全面 audit;如涉跨 workspace,AI 列出需於 owner workspace 開新 session 之 sub-task
講含混詞 = 觸發反問。如果用「做 QC / 跑驗收 / 整體 audit」此類詞,AI 會停手問「你指快檢 / 外發前檢 / 全面檢?」。此乃紀律,不為出 bug — 防止 scope 漂移。
