# Agent Public Squares v0.2.12 — Pre-release (Naming & Framing Build)

A pre-release on top of 0.2.11. Not a release pass. This build renames the product to **Agent Public Squares**, switches the user-facing vocabulary, and corrects the onboarding framing to match the shipped multi-peer model. No protocol or CLI behaviour changes — packets, lanes, ack, config keys, and flags are all unchanged.

## Status

- npm latest pre-release target: `@adamchanadam/aps@0.2.12`.
- npm package name is unchanged: `@adamchanadam/aps`. The abbreviation **APS** is identical for both the old and new product name, so there is zero install break and version continuity is preserved.
- Same Reliable Peer Handoff scope as 0.2.9–0.2.11 (Project Peers + Sent Status).

## Changed — product name

- **Product renamed `AI Public Squares` → `Agent Public Squares`** on all forward-facing surfaces: README, public docs (`docs/**`), the skill (`skills/aps/SKILL.md` + `references/setup-dialogue.md`), the CLI banner / help / generated route text (`bin/aps.js`), the protocol skeleton title (`resources/protocol/PROTOCOL.md`), and the QC reference card.
- **`AI Public Squares` remains a recognized LEGACY alias.** The skill still triggers on "教我用 AI Public Squares" / "set up AI Public Squares", and the GitHub RULE_PACKS route still lists the old name, so existing users and old phrasing keep working. Only the displayed product name changed.
- README carries a "前稱 AI Public Squares" sub-line for searchability and old-user recognition.

## Changed — vocabulary

- **Receive trigger `check Hub` → `check Drive`** in everything users see. `check Hub` is kept as a hidden recognized alias (skill body §6 + bin/aps.js route registration), so muscle memory still routes.
- **Concept word `Hub` / `Google Drive Hub` / `APS Hub` → 「共用 Drive 資料夾」** in user-facing prose.
- **Structural terms are deliberately unchanged**: the config key `hubRoot`, the flag `--hub-root`, path tokens `_hub/` / `<hub_root>` / `from_<agent>` / `_ack` / `outbox` / `packets`, the `APS Hub doctor` CLI token, and all PROTOCOL schema/paths. Renaming those would break existing `.aps/config.json` files and the protocol skeleton. The docs note once that the config key is still `hubRoot` internally so a user opening the file is not confused.

## Changed — onboarding framing (docs only)

- **The walkthrough was reframed from "two people" to the shipped multi-peer model**: one project can hand different parts to different collaborators; every handoff is still one-to-one and human-approved. The flow diagram now shows the sender choosing this handoff's recipient (e.g. Jay / Fanny) and labels the receiver as 「你揀嘅對象」; the 「讓對方加入」 section became 「可以不止一位」.
- This corrected a documentation QC gap: the entry page already framed multi-peer, but the walkthrough still read as strictly two-person, and a structure-only check passed both. A new release-check item (外發前檢 #9) now requires a content/product-model deep review across all user-facing pages after any rewrite.
- **The install section was NOT changed to over-promise.** `npx aps init` still asks for the counterpart's agent id; the "install only asks three things" humanize-onboarding change is deferred to 0.2.13.

## Changed — version wording

- Public surfaces (README, skill status line, maintainer page) no longer hard-code "0.2.8 / 0.2.9 candidate". They now say "前期測試版, 以 npm `@latest` 為準" and point to `npm view` / `npx aps --help` for the exact version, so docs no longer drift behind each bump.

## Repository

- **GitHub repo renamed `ai-public-squares` → `agent-public-squares`.** GitHub auto-redirects web + `git clone/fetch/push`, but NOT GitHub Pages project URLs (old Pages URL now 404s) nor Actions (this repo uses none). All in-file URLs (`package.json` repository/bugs/homepage, README links, `docs/index.html`, `bin/aps.js` route text) and the local git remote were updated to the new slug. The old repo name must not be reused or redirects break (per GitHub official rename docs).

## Migration for existing users

- Reinstall + refresh to pick up the new skill triggers and wording: `npm install --save-dev @adamchanadam/aps@latest`, then `npx aps upgrade`, then restart the AI tool. Until then, the installed skill only knows `check Hub`; both `check Drive` and `check Hub` route after upgrade.
- No Hub migration required. Existing shared-Drive folders, packets, outbox, ack, and `.aps/config.json` (including the `hubRoot` key) are untouched. Already-written `_hub/PROTOCOL.md` files in existing Hubs are not auto-renamed; only new installs get the new product name in the skeleton.

## Not done / deferred

- 0.2.13 humanize-onboarding (install asks only three things; doctor split into local-core vs peer health; publish prompts to choose/invite a peer when none; starter pack generated at invite time). This changes CLI behaviour and needs its own UAT.
- Deferred scope unchanged: true multi-agent, multi-recipient packets, `_notify`, `watch`, platform scheduling, Project Context Index runtime, Dropbox / OneDrive.
