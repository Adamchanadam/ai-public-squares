# AI Public Squares v0.2.10 — Pre-release (UAT Fix Build)

A pre-release patch on top of 0.2.9, cut to unblock real-machine UAT. Not a release pass.

## Status

- npm latest pre-release target: `@adamchanadam/aps@0.2.10`.
- Same Reliable Peer Handoff scope as 0.2.9 (Project Peers + Sent Status); this build only fixes a setup-validation bug found during real-machine UAT.

## Fixed

- **`aps init` rejected legitimate Hub root paths containing square brackets.** The placeholder guard treated any value containing `<`, `>`, `[`, or `]` as an unfilled placeholder. Real filesystem folders can legally be named with square brackets (e.g. a Google Drive path like `.../My Drive/[Project]/AI_Public_Squares`), so a Mac UAT was blocked at the Hub-root step. The guard now rejects only the documentation placeholder markers — `<...>` angle brackets and `...` ellipsis — and accepts square brackets. The snake_case fields (`--project`, `--agent-id`, `--other-agent-id`) are still bracket-rejected by `validateSnakeCase`, so identity values stay strict.
