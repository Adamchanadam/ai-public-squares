# Agent Public Squares v0.2.13 — 人性化上手 / Humanize Onboarding

建基於 0.2.12。今次令第一次安裝貼近一般人實際做法。**無破壞性改動**:npm 套件名 `@adamchanadam/aps`、簡寫 APS、既有設定全部照用。

## 新內容

- **三問安裝 / Three-question `init`** — `npx aps init` 只設定你自己一邊,只問三件事:共用 Drive 資料夾路徑、項目代號、你自己的 agent_id。唔再問對方、唔再問角色 A/B;對方改為日後用 `npx aps peer add` 另外邀請。
- **交辦事項明示契約 / Explicit items** — `--items "a;b"` 原文寫入封包標頭;收件方只讀標頭,永不從正文逆向猜。`revise` 保留上一版 items,`--clear-items` 清空。
- **共用資料夾預設名統一 `Agent_Public_Squares`** — 新安裝預設用此名;底線舊名 `AI_Public_Squares` 已由所有 user-facing 預設 / 示例移除。既有資料夾照用,無需搬遷。
- **新加入教學頁** — `peer add` 生成一段可轉發的邀請訊息,連去新的加入教學頁(內含三句安裝指令),讓非技術受邀者跟著裝好自己一邊。
- **doctor 本機核心 vs peer 分流** — 單人、剛裝、未邀請對方的項目 doctor 一樣通過(exit 0);對方通道健康只作參考、不影響退出碼。

## 相容性

- `AI Public Squares`(有空格)與 `check Hub` 仍是可識別的 legacy alias;正名為 `Agent Public Squares` / `check Drive`。
- 舊兩人 config(設有 `otherAgentId`)照舊運作,不強制轉新模型。**無需 Hub 遷移**:既有共用 Drive 資料夾、封包、lane、ack、`hubRoot` 全部不變。

## 安裝

```
npm install --save-dev @adamchanadam/aps@latest
npx aps init
```

既有項目升級:`npm install --save-dev @adamchanadam/aps@latest`,再 `npx aps upgrade`,然後重啟 AI 工具。
