# Agent Public Squares v0.2.13 — Pre-release (Humanize Onboarding Build)

A pre-release on top of 0.2.12. Not a release pass. This build makes first-time setup match how people actually work: `npx aps init` now sets up **only your own side** and asks **exactly three questions**, the counterpart is invited later as a separate step, and the action items on a handoff are an explicit sender declaration instead of being guessed from the body. The shipped multi-peer model (one project, many peers, every packet still one-to-one) is unchanged; this build removes the last places where setup still assumed a fixed two-person pair.

## 第一段 — CLI behaviour changes (committed earlier, S46)

`bin/aps.js`. These landed and were verified in the S46 build (isolated-Hub smoke 48/48 + `node --check`); listed here so the release note is the single changelog for 0.2.13.

- **Three-question interactive `init`.** Asks exactly three things in order — the shared Drive folder full local path, the project slug, and YOUR OWN agent_id — and sets up only your own side. It does **not** ask who the counterpart is and does **not** ask for a role (A/B). `otherAgentId` is saved as `null`. Role is auto-inferred solely to seed the bridge-pack fixture and is never used for authorization; when the project already exists with another active member, init shows a non-blocking "you look like the joiner" hint.
- **Non-interactive `init`** needs only the three core flags (`--hub-root` / `--project` / `--agent-id`). `--other-agent-id` and `--role A|B` are optional legacy two-person extras; supplying a counterpart still runs the old both-sides setup for backward compatibility.
- **`doctor` health split.** LOCAL-CORE checks decide the exit code (config, bridge, RULE_PACKS / PROJECT_INDEX registration, Drive root, `_hub` PROTOCOL/CHANGELOG, your own outbox/packets/ack, conflict scan); PEER health is purely informational and never flips the exit code. A solo, just-installed, not-yet-invited project **passes** (exit 0).
- **`publish` with no resolved recipient** prints actionable guidance (`未指定收件對象`, lists current peers, points to `--to` and `peer add`) instead of the old cryptic `Missing required values: --to`. The recipient reachability gate is conditional: an **explicit** `--to` to an unreachable peer hard-blocks; a config-default partner (no explicit `--to`) only warns and still sends, so the established two-person publish-then-invite flow is never hard-blocked.
- **Starter pack moved from `init` to `peer add`.** `npx aps peer add --agent-id <id>` creates the peer's lane/ack/card (provisional) and writes the starter pack to `_hub/starter-pack-<peer>.md`; `peer starter` regenerates it. `init` no longer writes a starter pack.
- **Items are an explicit sender contract.** `--items "a;b"` / `--items-file` are recorded verbatim into the packet frontmatter `items:` block; the reader/inbox reads items only from frontmatter and never reverse-parses them from the free-form body. `revise` carries prior items forward unless `--items` is given, or `--clear-items` empties them (`--items` and `--clear-items` are mutually exclusive).
- **Participation self-confirm.** Publishing or consuming as your configured agent marks only YOUR OWN peer card confirmed, never the counterpart's.
- The S45 inbox/notice **truncation fix** (single-ellipsis clipping) rides along.

## Naming unification — default shared-folder name `Agent_Public_Squares`

- **New default shared-folder name is `Agent_Public_Squares`.** Interactive `init` now guides the first person creating a hub to make a folder named `Agent_Public_Squares`, and a joiner to paste the shared folder the inviter sent. The underscore name `AI_Public_Squares` is removed from every user-facing default / example / prompt — the product is Agent Public Squares, so defaulting the folder to the old name was a confusion trap. (The product alias `AI Public Squares` with a space, and the `check Hub` trigger, remain recognized — only the underscore folder-name default was scrubbed.)
- Scrubbed `AI_Public_Squares` → `Agent_Public_Squares` across all shipped / onboarding surfaces: `bin/aps.js` init prompt + examples, README, `docs/index.html`, the onboarding walkthrough, `setup-dialogue.md`, and the demo Bridge Pack fixtures. The bridge-pack placeholder token was renamed to the neutral `<your_shared_drive_folder_absolute_path>` (kept in sync across `bin/aps.js` + both fixtures + `examples/README.md`).
- **Not changed (factual / deferred):** the design / plan docs under `docs/plans/*.md` still record the real runtime hub at `…\AI_Public_Squares\`, because that folder is not renamed (the local dev-folder / Drive-hub rename is a separate, no-live-session task). `init` already creates the folder you point it at (recursive, non-overwriting), but still cannot auto-locate Google Drive, so the user supplies the path.

## Onboarding rewrite — starter pack + joiner page

- **`starterPackContent()` rewritten.** `peer add` / `peer starter` now generate a short, forwardable written-Chinese invitation message — background, a few key steps, the must-match project code, the recipient's own name, and the shared-folder name (derived from `basename(hubRoot)`) — ending with a link to the new joiner guide. The old version's inline install commands, the Bridge Pack escape hatch, and the raw `publish` reply command were removed from the recipient-facing text.
- **New public joiner page `docs/guides/aps-join-invite.html`** (linked from the starter pack and the guides hub): for a non-technical person who was invited. Covers Google Drive share + offline-sync, a separate local work folder, AI-guided install **with the three actual commands** (so a fresh joiner whose AI does not yet have the APS skill can still install), the must-match project code, the success check, and short troubleshooting — plain written Chinese, emoji at key headings, no local `.md` links. Added to `docs/guides/index.html` and `dev/PROJECT_INDEX.md`.

## Migration for existing users

- Existing projects (with `.aps/config.json`) upgrade via `npm install --save-dev @adamchanadam/aps@latest` then `npx aps upgrade`, then restart the AI tool. `upgrade` backs up and refreshes the skill, updates the bridge and Handoff Kit registrations, runs a doctor pre-check, and never overwrites existing packets / outbox / ack / PROTOCOL.
- Old two-person configs (with `otherAgentId` set) keep working as a compatibility default; nothing forces them to the new model.
- No Hub migration required. Existing shared-Drive folders, packets, lanes, ack, and `hubRoot` are untouched.
