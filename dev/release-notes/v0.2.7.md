# AI Public Squares v0.2.7 — Pre-release

Reliable Pair hardening release.

## Added

- `npx aps upgrade` for existing APS projects. It refreshes APS skills, bridge files, and Handoff Kit registration without rebuilding Hub packets, outboxes, ack files, or protocol files.
- `publish --body-file` and `revise --body-file` for long generated packet bodies and safer shell handling.
- APS brand card for CLI help and AI skill flows.
- Functional emoji in CLI status output for easier scanning.

## Fixed

- `npx aps --help` is now Traditional Chinese and aligned with the rest of the command output.
- Packet notification wording no longer asks the receiver to use the sender's local Google Drive path. It tells the receiver to open their own APS-enabled project folder and run `check Hub`.
- `publish` packet scope now skips blank lines and Markdown headings before selecting the summary line.
- `init --dry-run` now says no files were changed instead of implying installation completed.
- APS skill guidance now separates APS packets from Agent Handoff Kit session handoff, reducing accidental handoff-route confusion.

## Updated

- Public HTML no longer previews unbuilt or deferred product capabilities.
- Runtime skill wording now emphasizes Reliable Pair first, sender confirmation before publish, receiver summary-first flow, missing-info follow-up, and consensus checks.
- Maintainer QC notes were updated to keep public promises aligned with current runtime behavior.
