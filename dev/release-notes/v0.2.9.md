This is a **pre-release / UAT enablement build**, not a release pass. Its purpose is to move npm `latest` to `0.2.9` so a second physical machine can install the candidate and run real cross-machine user acceptance testing. It is not production-ready.

## Status

- npm latest pre-release target: `@adamchanadam/aps@0.2.9`.
- Reliable Peer Handoff direction: one project may have multiple peers, but every packet stays single-recipient and human-approved.

## Included in this build

- **Project Peers + Sent Status**: `peers`, `peer add`, `peer starter`, `publish --to <agent_id>`, `inbox --from <agent_id>`, `inbox --all`, and `status --packet-id <id>`. A project can register multiple peers while each packet remains one-to-one. The old two-person `.aps/config.json` `otherAgentId` remains a compatibility view.
- **Codex skill description load-fix**: the skill frontmatter `description` was trimmed from 1052 to 701 characters and an unquoted-colon YAML issue was corrected, so Codex (which enforces a 1024-char load cap and rejects invalid YAML) now loads the official skill instead of silently falling back to a stale backup. Claude was unaffected, so this gap was Codex-specific.
- **Agent Public Squares route discovery**: `AI Public Squares` and `Agent Public Squares` are treated as the same brand / product-name variant; all three teach-me entries route into APS.
- **Starter pack preflight**: peer starter packs require the new peer to initialize Agent Handoff Kit before APS, so a clean folder is not left without a governance entry.
- **Receiver isolation / wrong-recipient guard**: `inbox --all` / `inbox --from` isolate packets by recipient; consuming a packet addressed to another peer is rejected; `status --packet-id` is sender-only and redirects receivers to `inbox`.

## Not done in this build (known boundaries)

- Human interactive UX UAT (a person driving a fresh Claude / Codex session through the natural-language flows) is **not yet done**; this build exists to enable it on a second machine.
- A true two-physical-machine Google Drive round-trip is **not yet done**; prior verification was same-machine or isolated-workspace.
- Class-C cross-workspace audit of the demo agent workspaces is **not yet run** for this candidate.
- Natural-language daily operation and recovery remain pre-release.
- Deferred (not in scope): true multi-agent platform, multi-recipient packets, group chat, file-based `_notify`, `aps watch`, desktop notifications, platform scheduling, Telegram bot auto-send, Project Context Index runtime, and Dropbox / OneDrive formal support. The core path requires no cloud API, OAuth, or developer project.
