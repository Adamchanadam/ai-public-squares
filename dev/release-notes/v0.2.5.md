# AI Public Squares v0.2.5 — Pre-release

This is a pre-release UX and governance hardening update for the APS CLI, public onboarding docs, bundled skill, and QC mechanism.

## Added

- Versioned Traditional Chinese install and doctor output.
- Connected APS banner, generic `agent_a` / `agent_b` setup defaults, and clearer guided setup field explanations.
- Handoff Kit APS route and project-index registration during `aps init`, without editing the managed `AGENTS.md` core.
- Copy-ready WhatsApp / Email notification text after `publish`.
- New public maintainer page that separates developer / QC material from the beginner route.
- Beginner walkthrough macro flow, capability boundary, one-sentence handoff path, sender / receiver preflight, missing-info path, and consensus-confirmation path.

## Fixed

- Rerunning `aps init` now updates `.aps/config.json`, local Bridge Pack, and starter pack after confirmation while preserving existing Hub protocol, lanes, ack files, and installed skill folders.
- `doctor` now reports status in Traditional Chinese and checks APS route / project-index registration.
- `publish` now accepts `--agent-id` as a `--from` alias and `--other-agent-id` as a `--to` alias, matching the saved-config override model.
- Public and package-facing examples avoid hardcoded person-specific defaults outside explicit examples or fixtures.

## Clarified

- APS remains pre-release.
- APS is not an automatic push-notification service; humans still send the generated notification through WhatsApp, Email, or another existing channel.
- CLI commands remain a verifiable fallback and troubleshooting route; the intended daily path is natural-language operation through an AI tool.
- Each real project still needs its own Google Drive path and sync verification.
